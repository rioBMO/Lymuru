# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.x     | ✅        |

Security fixes are released for the latest stable version. Older versions are not supported.

## Reporting a Vulnerability

We take security issues seriously. If you discover a security vulnerability in Lymuru, please report it **privately** — do not open a public issue.

**Contact:** [security@lymuru.app](mailto:security@lymuru.app)

Please include the following in your report:

- A detailed description of the vulnerability
- Steps to reproduce the issue
- Affected version(s)
- Any potential impact or exploit scenarios
- If possible, a suggested fix or mitigation

## What to Expect

1. **Acknowledgment** — You will receive a response within 48 hours confirming receipt of your report.
2. **Investigation** — We will investigate and determine the scope and severity of the issue.
3. **Resolution** — We will develop and test a fix. Critical vulnerabilities are prioritized.
4. **Disclosure** — After a fix is released, we will publicly acknowledge the issue and credit the reporter (unless you prefer to remain anonymous).

## Disclosure Policy

- Vulnerability reports are treated as confidential until a fix is released.
- We request that you do not publicly disclose the vulnerability before we have had a reasonable opportunity to address it.
- We will coordinate release dates with the reporter when appropriate.

## Scope

Security issues related to Lymuru's code, dependencies, and build process are in scope. Issues related to third-party services (Tidal, Qobuz, Amazon Music, Deezer, Telegram, LRCLIB) should be reported to those service providers directly.
